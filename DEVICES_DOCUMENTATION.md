## Система освещения

### Светильник (light_fixture)
**Тип устройства:** `light_fixture`  
**Система:** `lighting`  
**Иконка:** 💡 Lightbulb

#### Описание
Светильник для внутреннего освещения помещений. Поддерживает управление яркостью, может быть объединен в группы для централизованного управления. Может иметь статус исправности для мониторинга работоспособности.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/light_fixture/{deviceId}/State`  
**Управление:** `building/floor{N}/room{roomId}/light_fixture/{deviceId}/Set`

#### Структура данных

**State (чтение):**
```typescript
{
  lightLevel: number,        // Текущая яркость 0-100%
  lightGroup: string,        // Группа освещения (авто по помещению или вручную)
  lightStatus: 'ok' | 'error' // Статус исправности
}
```

**Set (управление):**
```typescript
{
  lightLevel?: number,       // Установить яркость 0-100%
  lightGroup?: string        // Изменить группу освещения
}
```

#### Группировка светильников
- **Автоматическая группировка** - по умолчанию все светильники в помещении формируют одну группу
- **Ручная группировка** - можно создавать произвольные группы и назначать светильники
- **Управление группой** - изменение яркости применяется ко всей группе одновременно
- **Примеры групп:** `room_101`, `corridor_north`, `office_zone_A`

#### Особенности размещения
- Обычно размещается равномерно по всему помещению
- Доступна функция автоматической расстановки с равномерным распределением
- Светильники привязываются друг к другу при перемещении (виртуальные направляющие)
- Масштаб по умолчанию настраивается глобально для помещения

#### Интеграция MQTT

**Команды управления:**
- Установить яркость отдельного светильника
- Установить яркость группы светильников
- Включить/выключить (яркость 0/100)

**События и уведомления:**
- Изменение яркости
- Изменение статуса исправности
- Изменение группы

#### Примеры использования

**Пример 1: Установка яркости одного светильника**
```
Topic: building/floor1/room101/light_fixture/light_001/Set
Payload: {"lightLevel": 75}
```

**Пример 2: Управление группой светильников**
```
Для всех устройств с lightGroup="office_zone_A"
Topic: building/floor1/room101/light_fixture/{deviceId}/Set
Payload: {"lightLevel": 50}
```

**Пример 3: Мониторинг состояния**
```
Topic: building/floor1/room101/light_fixture/light_001/State
Payload: {
  "lightLevel": 75,
  "lightGroup": "room_101",
  "lightStatus": "ok"
}
```

### Уличный светильник (street_light)
**Тип устройства:** `street_light`  
**Система:** `lighting`  
**Иконка:** 💡 Lightbulb

#### Описание
Светильник для наружного освещения территории, входных зон, парковок. Функционал аналогичен внутреннему светильнику, но предназначен для размещения на внешних планах.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/outdoor/street_light/{deviceId}/State`  
**Управление:** `building/floor{N}/outdoor/street_light/{deviceId}/Set`

#### Структура данных
Аналогична светильнику (см. light_fixture)

#### Особенности размещения
- Размещается на планах внешних территорий
- Может быть привязан к таймерам (включение в сумерках)
- Группировка по зонам освещения (входная группа, парковка, периметр)

## Система контроля доступа (СКУД)

### Дверь (door)
**Тип устройства:** `door`  
**Система:** `access_control`  
**Иконка:** 🚪 DoorOpen

#### Описание
Дверь с электронным управлением доступом. Контролирует вход/выход, поддерживает различные режимы работы для гибкого управления доступом.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/door/{deviceId}/State`  
**Управление:** `building/floor{N}/room{roomId}/door/{deviceId}/Set`

#### Структура данных

**State (чтение):**
```typescript
{
  isOpen: boolean,         // Открыта (true) / Закрыта (false)
  isOnControl: boolean,    // На контроле - требуется авторизация
  isClosed: boolean,       // Полностью закрыта
  mode: 'office' | 'control' | 'open' | 'closed' // Текущий режим работы
}
```

**Set (управление):**
```typescript
{
  mode: 'office' | 'control' | 'open' | 'closed' // Установить режим работы
}
```

#### Режимы работы
- **office** - Офисный режим: свободный вход, контроль выхода
- **control** - Режим контроля: требуется авторизация для входа и выхода
- **open** - Открыто: свободный проход без контроля
- **closed** - Закрыто: проход заблокирован

#### Особенности размещения
- Двери можно размещать на границах помещений (края стен)
- По умолчанию дверь поворачивается на 90° при добавлении
- Дверь размещается по центру левой стены помещения

#### Интеграция MQTT

**Команды управления:**
- Изменить режим работы двери
- Временное открытие (разовая команда)

**События и уведомления:**
- Изменение состояния (открыто/закрыто)
- Несанкционированное открытие
- Смена режима работы
- События прохода (с идентификатором пользователя)

#### Примеры использования

**Пример 1: Установка офисного режима**
```
Topic: building/floor1/room101/door/door_001/Set
Payload: {"mode": "office"}
```

**Пример 2: Мониторинг состояния**
```
Topic: building/floor1/room101/door/door_001/State
Payload: {
  "isOpen": false,
  "isOnControl": true,
  "isClosed": true,
  "mode": "control"
}
```

**Тип устройства:** `turnstile`  
**Система:** `access_control`  
**Иконка:** 🔷 Hexagon

#### Описание
Полноростовой турникет для контроля доступа на объект. Используется на входах в здание, обеспечивает пропуск только авторизованных лиц.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/entrance/turnstile/{deviceId}/State`  
**Управление:** `building/floor{N}/entrance/turnstile/{deviceId}/Set`

#### Структура данных
Структура данных и режимы работы аналогичны двери (см. door)

#### Особенности
- Обычно размещается в холлах, входных зонах
- Всегда требует авторизации (режим control по умолчанию)
- Фиксирует направление прохода (вход/выход)

## Система видеонаблюдения

### Видеокамера (camera)
**Тип устройства:** `camera`  
**Система:** `video`  
**Иконка:** 📹 Video

#### Описание
Видеокамера для наблюдения и записи событий в помещениях и на территории. Предоставляет доступ к видеопотоку в реальном времени.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/camera/{deviceId}/State`  
**Управление:** `building/floor{N}/room{roomId}/camera/{deviceId}/Set`

#### Структура данных

**State (чтение):**
```typescript
{
  streamUrl: string,       // URL потока видео (RTSP, HLS, WebRTC)
  recording: boolean,      // Статус записи (включена/выключена)
  status: 'online' | 'offline' | 'warning'
}
```

**Set (управление):**
```typescript
{
  recording?: boolean      // Включить/выключить запись
}
```

#### Особенности размещения
- Размещается на стенах или потолке
- Поворот камеры указывает направление обзора
- Рекомендуется размещать в углах для максимального охвата

#### Интеграция MQTT

**Команды управления:**
- Включить/выключить запись
- Запросить снимок (snapshot)

**События и уведомления:**
- Потеря видеосигнала
- Начало/окончание записи
- Обнаружение движения (если поддерживается)

#### URL потока видео
Поддерживаемые форматы:
- `rtsp://192.168.1.100:554/stream1` - RTSP поток
- `https://video.example.com/camera1/stream.m3u8` - HLS поток
- `wss://video.example.com/camera1` - WebRTC

#### Примеры использования

**Пример 1: Чтение состояния камеры**
```
Topic: building/floor1/room101/camera/cam_001/State
Payload: {
  "streamUrl": "rtsp://192.168.1.100:554/stream1",
  "recording": true,
  "status": "online"
}
```

**Пример 2: Включение записи**
```
Topic: building/floor1/room101/camera/cam_001/Set
Payload: {"recording": true}
```

## Система вентиляции и кондиционирования (HVAC)

### Настенный кондиционер (wall_ac)
**Тип устройства:** `wall_ac`  
**Система:** `hvac`  
**Иконка:** ❄️ Wind

#### Описание
Настенный блок кондиционера для охлаждения/нагрева помещений. Устанавливается на стене, имеет полный набор функций климат-контроля.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/wall_ac/{deviceId}/State`  
**Управление:** `building/floor{N}/room{roomId}/wall_ac/{deviceId}/Set`

#### Структура данных

**State (чтение):**
```typescript
{
  isPowerOn: boolean,                          // Включен/выключен
  fanMode: 'auto' | 'low' | 'medium' | 'high', // Режим вентилятора
  workMode: 'cool' | 'heat' | 'fan' | 'dry' | 'auto', // Режим работы
  targetTemperature: number,                   // Уставка температуры (°C)
  currentTemperature: number,                  // Текущая температура (°C)
  error?: string                               // Код ошибки (если есть)
}
```

**Set (управление):**
```typescript
{
  isPowerOn?: boolean,                         // Вкл/выкл
  fanMode?: 'auto' | 'low' | 'medium' | 'high',
  workMode?: 'cool' | 'heat' | 'fan' | 'dry' | 'auto',
  targetTemperature?: number                   // Установить уставку (16-30°C)
}
```

#### Режимы работы
- **cool** - Охлаждение
- **heat** - Обогрев
- **fan** - Вентиляция (без изменения температуры)
- **dry** - Осушение
- **auto** - Автоматический выбор режима

#### Режимы вентилятора
- **auto** - Автоматическая регулировка скорости
- **low** - Низкая скорость (тихий режим)
- **medium** - Средняя скорость
- **high** - Высокая скорость (быстрое охлаждение/нагрев)

#### Особенности размещения
- Настенный блок устанавливается на стене
- Обычно на высоте 2-2.5 м от пола
- Направление потока воздуха зависит от поворота устройства

#### Коды ошибок
- `E1` - Ошибка датчика температуры
- `E2` - Утечка хладагента
- `E3` - Перегрев компрессора
- `E4` - Ошибка связи с внешним блоком

#### Примеры использования

**Пример 1: Включение охлаждения**
```
Topic: building/floor1/room101/wall_ac/ac_001/Set
Payload: {
  "isPowerOn": true,
  "workMode": "cool",
  "targetTemperature": 22,
  "fanMode": "auto"
}
```

**Пример 2: Мониторинг состояния**
```
Topic: building/floor1/room101/wall_ac/ac_001/State
Payload: {
  "isPowerOn": true,
  "fanMode": "medium",
  "workMode": "cool",
  "targetTemperature": 22,
  "currentTemperature": 24,
  "error": null
}
```

### Кассетный кондиционер (cassette_ac)
**Тип устройства:** `cassette_ac`  
**Система:** `hvac`  
**Иконка:** ❄️ Wind

#### Описание
Кассетный блок кондиционера, встраиваемый в подвесной потолок. Обеспечивает равномерное распределение воздуха по всему помещению.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/cassette_ac/{deviceId}/State`  
**Управление:** `building/floor{N}/room{roomId}/cassette_ac/{deviceId}/Set`

#### Структура данных
Структура данных, режимы работы и управление полностью аналогичны настенному кондиционеру (см. wall_ac)

#### Особенности размещения
- Встраивается в подвесной потолок
- Может занимать практически любое место на потолке
- Обеспечивает равномерное распределение воздуха (4 направления)
- Менее заметен визуально, чем настенный блок

## Система отопления

### Батарея отопления (radiator)
**Тип устройства:** `radiator`  
**Система:** `heating`  
**Иконка:** 💧 Droplet

#### Описание
Радиатор отопления с терморегулирующей головкой для поддержания комфортной температуры в помещении.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/radiator/{deviceId}/State`  
**Управление:** `building/floor{N}/room{roomId}/radiator/{deviceId}/Set`

#### Структура данных

**State (чтение):**
```typescript
{
  thermostatSetting: number,    // Текущая уставка термоголовки (0-100%)
  currentTemperature: number,   // Температура теплоносителя (°C)
  valveOpen: number             // Степень открытия клапана (0-100%)
}
```

**Set (управление):**
```typescript
{
  thermostatSetting: number     // Установить уставку термоголовки (0-100%)
}
```

#### Особенности размещения
- Размещается вдоль внешних стен под окнами
- По умолчанию выбран при создании нового помещения
- Количество секций зависит от площади помещения

#### Управление термоголовкой
- **0%** - Клапан закрыт, нагрев отключен
- **25%** - Минимальный нагрев (экономичный режим)
- **50%** - Средний уровень нагрева
- **75%** - Интенсивный нагрев
- **100%** - Максимальный нагрев

#### Примеры использования

**Пример 1: Установка уставки**
```
Topic: building/floor1/room101/radiator/rad_001/Set
Payload: {"thermostatSetting": 60}
```

**Пример 2: Мониторинг состояния**
```
Topic: building/floor1/room101/radiator/rad_001/State
Payload: {
  "thermostatSetting": 60,
  "currentTemperature": 65,
  "valveOpen": 45
}
```

### Тепловая завеса (air_curtain)
**Тип устройства:** `air_curtain`  
**Система:** `heating`  
**Иконка:** ❄️ Wind

#### Описание
Тепловая завеса для защиты от холодного воздуха при открывании дверей. Создает воздушный барьер, предотвращающий потери тепла.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/air_curtain/{deviceId}/State`  
**Управление:** `building/floor{N}/room{roomId}/air_curtain/{deviceId}/Set`

#### Структура данных

**State (чтение):**
```typescript
{
  curtainPowerOn: boolean,        // Включена/выключена
  curtainTemperature: number,     // Уставка температуры воздуха (°C)
  fanSpeed: 'low' | 'medium' | 'high', // Скорость вентилятора
  heaterOn: boolean               // Статус нагревателя
}
```

**Set (управление):**
```typescript
{
  curtainPowerOn?: boolean,       // Вкл/выкл
  curtainTemperature?: number,    // Установить температуру (20-40°C)
  fanSpeed?: 'low' | 'medium' | 'high'
}
```

#### Особенности размещения
- Устанавливается над дверными проемами
- Обычно автоматически включается при открытии двери
- Ширина должна соответствовать ширине дверного проема

#### Режимы работы
- **Только вентилятор** - Без нагрева (летний режим)
- **Нагрев + вентилятор** - Полноценная тепловая завеса (зимний режим)

#### Примеры использования

**Пример 1: Включение тепловой завесы**
```
Topic: building/floor1/entrance/air_curtain/curtain_001/Set
Payload: {
  "curtainPowerOn": true,
  "curtainTemperature": 30,
  "fanSpeed": "medium"
}
```

**Пример 2: Мониторинг состояния**
```
Topic: building/floor1/entrance/air_curtain/curtain_001/State
Payload: {
  "curtainPowerOn": true,
  "curtainTemperature": 30,
  "fanSpeed": "medium",
  "heaterOn": true
}
```

## Система датчиков

### Датчик температуры (temperature_sensor)
**Тип устройства:** `temperature_sensor`  
**Система:** `sensors`  
**Иконка:** 🌡️ Thermometer

#### Описание
Датчик для измерения температуры воздуха в помещении. Используется для мониторинга и автоматического управления системами отопления и кондиционирования.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/temperature_sensor/{deviceId}/State`

**⚠️ Примечание:** Датчики работают только в режиме чтения, управляющие команды не поддерживаются.

#### Структура данных

**State (чтение):**
```typescript
{
  temperature: number,      // Температура воздуха (°C)
  humidity?: number,        // Влажность (%) - если датчик поддерживает
  timestamp: number         // Время измерения (Unix timestamp)
}
```

#### Особенности размещения
- Размещается на стене на высоте около 150 см
- Не должен находиться рядом с источниками тепла/холода
- Избегать попадания прямых солнечных лучей
- Не размещать рядом с дверьми и окнами

#### Интеграция с другими системами
- Автоматическое управление кондиционерами
- Регулирование отопления
- Системы вентиляции
- Пожарная сигнализация (обнаружение перегрева)

#### Примеры использования

**Пример 1: Чтение данных**
```
Topic: building/floor1/room101/temperature_sensor/temp_001/State
Payload: {
  "temperature": 22.5,
  "humidity": 45,
  "timestamp": 1710172800
}
```

**Пример 2: Интеграция с системой отопления**
```
При temperature < 20°C → включить радиатор
При temperature > 24°C → уменьшить нагрев
```

### Датчик CO2 (co2_sensor)
**Тип устройства:** `co2_sensor`  
**Система:** `sensors`  
**Иконка:** 📊 Activity

#### Описание
Датчик для измерения концентрации углекислого газа (CO2) в воздухе. Используется для контроля качества воздуха и управления системами вентиляции.

#### MQTT топики
**Чтение состояния:** `building/floor{N}/room{roomId}/co2_sensor/{deviceId}/State`

**⚠️ Примечание:** Датчики работают только в режиме чтения, управляющие команды не поддерживаются.

#### Структура данных

**State (чтение):**
```typescript
{
  co2: number,              // Концентрация CO2 (ppm - parts per million)
  temperature?: number,     // Температура (°C) - если датчик поддерживает
  humidity?: number,        // Влажность (%) - если датчик поддерживает
  timestamp: number         // Время измерения (Unix timestamp)
}
```

#### Нормативные значения CO2
- **< 800 ppm** - Отличное качество воздуха
- **800-1000 ppm** - Хорошее качество воздуха
- **1000-1500 ppm** - Удовлетворительное качество, рекомендуется проветривание
- **1500-2000 ppm** - Плохое качество воздуха, требуется усиленная вентиляция
- **> 2000 ppm** - Недопустимый уровень, немедленная вентиляция

#### Особенности размещения
- Размещается на высоте дыхания (120-180 см)
- В местах с хорошей циркуляцией воздуха
- Не размещать непосредственно у вентиляционных отверстий
- Оптимально - в центре помещения или около рабочих мест

#### Интеграция с другими системами
- Автоматическое управление вентиляцией
- Контроль работы кондиционеров
- Системы "умного здания"
- Предупреждения о необходимости проветривания

#### Примеры использования

**Пример 1: Чтение данных**
```
Topic: building/floor1/room101/co2_sensor/co2_001/State
Payload: {
  "co2": 850,
  "temperature": 22.0,
  "humidity": 45,
  "timestamp": 1710172800
}
```

**Пример 2: Автоматическое управление вентиляцией**
```
При co2 > 1000 ppm → увеличить скорость вентиляции
При co2 > 1500 ppm → максимальная вентиляция + уведомление
При co2 < 800 ppm → нормальный режим вентиляции
```

## Общие рекомендации

### Масштабирование устройств
- Все устройства поддерживают масштабирование от 50% до 200%
- Глобальный масштаб по умолчанию настраивается в панели помещения
- Можно изменять масштаб для множественного выделения устройств

### Поворот устройств
- Все устройства можно поворачивать с шагом 15° (0-360°)
- Быстрые кнопки: 0°, 90°, 180°, 270°
- Поворот важен для дверей, камер и направленных устройств (кондиционеры, тепловые завесы)

### Множественное выделение
- Используйте Ctrl+клик для выделения нескольких устройств
- Можно одновременно изменять масштаб и поворот группы устройств
- Полезно для синхронной настройки однотипных устройств

### Система видимости
- Каждый тип устройства можно скрыть на плане
- Скрытые устройства не удаляются, только не отображаются
- Настройки видимости сохраняются между сеансами
- Полезно для фокусировки на конкретной системе

### Группировка устройств в фильтрах (версия 2.1)
Для удобства управления видимостью устройства объединены в логические группы:

**Группы устройств:**
1. **Освещение** (💡 Lightbulb)
   - Светильник (light_fixture)
   - Уличный светильник (street_light)

2. **Двери и ворота** (🚪 DoorOpen)
   - Дверь (door)
   - Ворота (gate)

3. **Турникеты** (🔷 Hexagon)
   - Турникет (turnstile)
   - Тумбовый турникет (tripod_turnstile)

4. **Шлагбаумы** (⬇️ Minimize2)
   - Шлагбаум (barrier)

5. **Видеокамеры** (📹 Video)
   - Видеокамера (camera)

6. **Проекторы** (📽️ Projector)
   - Проектор (projector)

7. **Телевизоры** (📺 Tv)
   - Телевизор (tv)

8. **Кондиционеры** (❄️ Wind)
   - Настенный кондиционер (wall_ac)
   - Кассетный кондиционер (cassette_ac)

9. **Отопление** (💧 Droplet)
   - Батарея отопления (radiator)
   - Тепловая завеса (air_curtain)

    - Датчик температуры (temperature_sensor)

**Поведение групп:**
- Клик по иконке группы переключает видимость всех устройств в группе
- Активная иконка (синяя) - все устройства группы видимы
- Иконка с желтым индикатором - часть устройств группы видима
- Перечеркнутая иконка (красная линия) - все устройства группы скрыты
- Это позволяет управлять 17 типами устройств через 10 компактных пиктограмм

## MQTT протокол

### Структура топиков

**Общий формат:**
```
building/floor{N}/room{roomId}/{deviceType}/{deviceId}/{command}
```

**Команды:**
- `State` - Чтение текущего состояния устройства
- `Set` - Управление устройством (установка параметров)

**Примеры:**
```
building/floor1/room101/light_fixture/light_001/State
building/floor1/room101/light_fixture/light_001/Set
building/floor2/room205/wall_ac/ac_003/State
building/floor2/room205/wall_ac/ac_003/Set
```

### Формат сообщений

**JSON формат:**
Все сообщения передаются в формате JSON:
```json
{
  "parameter1": value1,
  "parameter2": value2
}
```

**QoS уровни:**
- **QoS 0** - Датчики (не критично потерять одно показание)
- **QoS 1** - Мониторинг состояний
- **QoS 2** - Управляющие команды (критично)

### Команды управления

**Стандартные команды для всех управляемых устройств:**

1. **Включить/выключить** (где применимо)
```json
{"isPowerOn": true}
{"curtainPowerOn": false}
```

2. **Установить режим работы** (СКУД, HVAC)
```json
{"mode": "control"}
{"workMode": "cool"}
```

3. **Установить параметр**
```json
{"lightLevel": 75}
{"targetTemperature": 22}
{"thermostatSetting": 60}
```

### События и уведомления

**Автоматические публикации:**
- Изменение состояния → публикация в `/State`
- Ошибка → публикация с полем `error`
- Предупреждение → публикация с полем `warning`

**Частота публикаций:**
- Датчики: каждые 30-60 секунд
- Состояния устройств: при изменении + каждые 5 минут
- Критические события: немедленно

### Retain флаг
Используется для сохранения последнего состояния:
- Все `/State` топики - retain=true
- Управляющие `/Set` команды - retain=false

## Интеграция систем

### Сценарии автоматизации

**Пример 1: Энергосбережение**
```
Условие: НЕТ движения 30 минут + CO2 < 800 ppm
Действия:
- Выключить 50% светильников
- Снизить температуру кондиционера на 2°C
- Уменьшить вентиляцию
```

**Пример 2: Комфортный климат**
```
Условие: Температура < 20°C
Действия:
- Увеличить уставку радиаторов до 75%
- Включить тепловые завесы у входов
- Отправить уведомление диспетчеру
```

**Пример 3: Контроль доступа**
```
Условие: 18:00 - окончание рабочего дня
Действия:
- Перевести все двери в режим "control"
- Включить все камеры на запись
- Выключить 80% освещения
- Снизить температуру до 18°C
```

**Пример 4: Качество воздуха**
```
Условие: CO2 > 1000 ppm
Действия:
- Увеличить скорость вентиляции до максимума
- Отправить уведомление о необходимости проветривания
- Логировать событие для анализа
```

### Группировка устройств

**По помещениям:**
- Все устройства в помещении имеют одинаковый `roomId` в топике
- Позволяет управлять всем помещением одной командой

**По системам:**
- Фильтрация по типу системы (lighting, heating, hvac и т.д.)
- Централизованное управление одной системой

**По группам (освещение):**
- Произвольные группы для гибкого управления
- Несколько групп могут пересекаться

- Логирование всех управляющих команд
