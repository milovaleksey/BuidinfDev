# 🔧 Исправление 403 Forbidden

## Проблема

После исправления CORS появилась новая ошибка:
```
POST http://10.81.0.60:8080/api/auth/login 403 (Forbidden)
```

## Причина

**Дублирование префикса `/api` в путях:**

В `application.properties` установлен:
```properties
server.servlet.context-path=/api
```

Это означает, что Spring автоматически добавляет `/api` ко всем путям.

Но в контроллерах тоже был указан `/api`:
```java
@RequestMapping("/api/auth")  // ❌ Неправильно
```

**Итоговый путь получался:** `/api` + `/api/auth` = `/api/api/auth/login` ❌

**А фронтенд обращался к:** `/api/auth/login` ✅

## Решение

Убран префикс `/api` из всех контроллеров, так как он уже добавляется через `context-path`.

### Исправленные файлы:

1. ✅ **AuthController.java** - `/api/auth` → `/auth`
2. ✅ **BuildingController.java** - уже было `/buildings` (не менялось)
3. ✅ **FloorController.java** - `/api/floors` → `/floors`
4. ✅ **RoomController.java** - `/api/rooms` → `/rooms`
5. ✅ **DeviceController.java** - `/api/devices` → `/devices`
6. ✅ **AuditLogController.java** - `/api/audit` → `/audit`
7. ✅ **ProxyController.java** - `/api/nodered` → `/nodered`

## Troubleshooting

### Проблема: Всё ещё 403 Forbidden

**Решение 1:** Убедитесь, что бэкенд перезапущен
```bash
ps aux | grep java
# Если есть старый процесс, остановите его:
pkill -f spring-boot
cd backend && ./start.sh
```

**Решение 2:** Проверьте логи
```bash
cd backend
tail -f nohup.out | grep -E "(ERROR|WARN|403)"
```

**Решение 3:** Проверьте что пользователи в БД
```bash
cd backend
./check-db.sh
```

### Проблема: 401 Unauthorized

Это означает что авторизация работает, но неверные учетные данные.

Проверьте:
```bash
cd backend
sudo -u postgres psql -d building_management -c "SELECT username, email FROM users;"
```

Если пользователей нет:
```bash
./load-demo-data.sh
```

### Проблема: Страница не загружается

**Решение:** Проверьте что фронтенд запущен
```bash
cd frontend
npm run dev
```

Откройте `http://10.81.0.60:5173` (не localhost!)
